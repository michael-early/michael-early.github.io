Create `.github/workflows/` directory and `portfolio-agent.yml` with the following exact structure (all decisions are locked — do not deviate):

Trigger:
- `on: issues: types: [opened]` only. No other event types.

Job `agent`:
- `runs-on: ubuntu-latest`
- Job-level `if: github.event.issue.user.login == github.repository_owner` — use this field (issue author), NOT `github.actor`. Do not use `${{ }}` wrapper (optional per GitHub docs but inconsistent style).
- `permissions: issues: read, contents: read` — minimum for Phase 1; do not set `contents: write`.

Steps (in this exact order):
1. `name: Skip if bot actor` — env: `ACTOR: ${{ github.actor }}`. Bash: `if [ "$ACTOR" = "github-actions[bot]" ]; then echo "Skipping: triggered by bot actor"; exit 0; fi`. This step must be FIRST.
2. `name: Assert secrets wired` — env: `ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}`. Bash: `if [ -z "$ANTHROPIC_API_KEY" ]; then echo "ERROR: ANTHROPIC_API_KEY secret is not set"; exit 1; fi && echo "Secrets check passed"`.
3. `uses: actions/checkout@v4` — no additional config needed for Phase 1.
4. `name: Run agent stub` — env: `ISSUE_NUMBER: ${{ github.event.issue.number }}`, `ISSUE_TITLE: ${{ github.event.issue.title }}`, `ISSUE_BODY: ${{ github.event.issue.body }}`. Run: `python agent/run.py`.

Anti-patterns to avoid (per RESEARCH.md):
- Do NOT combine the owner check and bot guard into a single `if:`.
- Do NOT use `github.actor` for the owner check.
- Do NOT hardcode the owner username — use `github.repository_owner`.
- Do NOT use `set-env` (deprecated since 2020); use step-level `env:` blocks.
- Do NOT use `actions/checkout@v2` or `@v3`; use `@v4`.
